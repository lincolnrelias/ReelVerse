# REELVERSE — Cursor Development Specification
# Fase 1: Ciclo Completo de Funcionalidade Básica (YouTube Shorts Only)

## CONTEXTO DO PROJETO

ReelVerse é uma plataforma web que recebe o link de um YouTube Short e entrega uma análise de engenharia reversa completa do vídeo. O usuário cola o link, o sistema extrai dados, processa com IA e exibe um relatório visual com insights acionáveis.

**Escopo desta fase:** Fechar o ciclo funcional completo (input → extração → processamento → análise IA → exibição) com YouTube Shorts apenas. Sem autenticação, sem planos, sem histórico persistente por usuário. O objetivo é ter o pipeline end-to-end funcionando.

  videoUrl: text('video_url').notNull(),

## API ENDPOINTS (apps/api)

### POST /api/analysis
Cria uma nova análise. Valida o URL (deve ser youtube.com/shorts/* ou youtu.be/*), cria registro no banco com status "pending" e enfileira o job no BullMQ.

**Request body:**
```json
{ "videoUrl": "https://www.youtube.com/shorts/VIDEO_ID" }
```

**Response (201):**
```json
{ "id": "uuid", "status": "pending", "createdAt": "ISO8601" }
```

**Validação do URL (Zod):**
```typescript
const youtubeShortRegex = /^https?:\/\/(www\.)?(youtube\.com\/shorts\/[\w-]+|youtu\.be\/[\w-]+)/;

const createAnalysisSchema = z.object({
  videoUrl: z.string().url().regex(youtubeShortRegex, 'URL deve ser um YouTube Short válido'),
});
```

### GET /api/analysis/:id
Retorna o status atual da análise. O frontend faz polling neste endpoint.

**Response (200):**
```json
{
  "id": "uuid",
  "status": "analyzing",
  "videoMeta": { ... },       // disponível após status "extracting" concluir
  "result": null,             // null até status "completed"
  "processingTimeMs": null,
  "createdAt": "ISO8601",
  "completedAt": null
}
```

Quando status="completed", result contém o AnalysisResult completo.
Quando status="failed", errorMessage contém a descrição do erro.

### GET /api/analysis/:id/cache-check (otimização)
Antes de criar uma análise, o frontend pode verificar se o mesmo URL já foi analisado nas últimas 24h.

**Query:** `?videoUrl=encoded_url`
**Response:** `{ "cached": true, "analysisId": "uuid" }` ou `{ "cached": false }`

### Etapa 1: Extração (status: "extracting")

```typescript
// Usa yt-dlp para extrair metadados e baixar o vídeo
// Comando: yt-dlp --dump-json URL → metadados JSON
// Comando: yt-dlp -f "bestvideo[height<=1080]+bestaudio/best[height<=1080]" -o TEMP_PATH URL → download

async function extractVideo(videoUrl: string, tempDir: string): Promise<{ videoPath: string; meta: VideoMeta }> {
  // 1. Extrair metadados com --dump-json
  const metaJson = await execCommand('yt-dlp', ['--dump-json', videoUrl]);
  const raw = JSON.parse(metaJson);
  
  const meta: VideoMeta = {
    title: raw.title || '',
    description: raw.description || '',
    channelName: raw.channel || raw.uploader || '',
    channelUrl: raw.channel_url || '',
    publishDate: raw.upload_date || '',
    duration: raw.duration || 0,
    viewCount: raw.view_count || 0,
    likeCount: raw.like_count || 0,
    commentCount: raw.comment_count || 0,
    hashtags: extractHashtags(raw.description),
    thumbnailUrl: raw.thumbnail || '',
    videoId: raw.id || '',
  };
  
  // 2. Baixar vídeo
  const videoPath = path.join(tempDir, `${meta.videoId}.mp4`);
  await execCommand('yt-dlp', [
    '-f', 'bestvideo[height<=720]+bestaudio/best[height<=720]',
    '--merge-output-format', 'mp4',
    '-o', videoPath,
    videoUrl,
  ]);
  
  return { videoPath, meta };
}
```

### Etapa 2: Processamento de Vídeo com FFmpeg (status: "processing_video")

```typescript
async function processVideo(videoPath: string, tempDir: string): Promise<{
  audioPath: string;
  framesDir: string;
  cutTimestamps: number[];
  duration: number;
}> {
  const audioPath = path.join(tempDir, 'audio.mp3');
  const framesDir = path.join(tempDir, 'frames');
  await fs.mkdir(framesDir, { recursive: true });
  
  // 1. Extrair áudio
  await execCommand('ffmpeg', [
    '-i', videoPath,
    '-vn', '-acodec', 'libmp3lame', '-q:a', '4',
    audioPath,
  ]);
  
  // 2. Extrair frames (1 por segundo)
  await execCommand('ffmpeg', [
    '-i', videoPath,
    '-vf', 'fps=1',
    '-q:v', '3',
    path.join(framesDir, 'frame_%03d.jpg'),
  ]);
  
  // 3. Detectar cortes (scene change detection)
  const sceneOutput = await execCommand('ffmpeg', [
    '-i', videoPath,
    '-filter:v', "select='gt(scene,0.3)',showinfo",
    '-f', 'null', '-',
  ], { captureStderr: true });
  
  const cutTimestamps = parseSceneDetection(sceneOutput);
  
  return { audioPath, framesDir, cutTimestamps, duration: /* from ffprobe */ };
}
```

```typescript
import OpenAI from 'openai';

async function transcribeAudio(audioPath: string): Promise<TranscriptionData> {
  const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });
  
  const file = fs.createReadStream(audioPath);
  const response = await openai.audio.transcriptions.create({
    model: 'whisper-1',
    file,
    response_format: 'verbose_json',
    timestamp_granularities: ['segment'],
  });
  
  return {
    fullText: response.text,
    segments: (response.segments || []).map(s => ({
      start: s.start,
      end: s.end,
      text: s.text.trim(),
    })),
    language: response.language || 'unknown',
    hasVoiceover: response.text.trim().length > 10,
  };
}
```

### Etapa 4: Análise Consolidada com Claude (status: "analyzing")

```typescript
import Anthropic from '@anthropic-ai/sdk';

async function analyzeWithAI(params: {
  meta: VideoMeta;
  transcription: TranscriptionData;
  frameAnalysis: FrameAnalysisData;
}): Promise<AnalysisResult> {
  const anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });
  
  // Montar o prompt com todos os dados coletados
  const prompt = buildAnalysisPrompt(params);
  
  const response = await anthropic.messages.create({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 4096,
    messages: [{ role: 'user', content: prompt }],
    system: ANALYSIS_SYSTEM_PROMPT,
  });
  
  const text = response.content[0].type === 'text' ? response.content[0].text : '';
  
  // Extrair JSON do response (pode vir com markdown code fences)
  const jsonStr = text.replace(/```json\n?|\n?```/g, '').trim();
  const parsed = JSON.parse(jsonStr);
  
  // Validar com Zod schema
  return analysisResultSchema.parse(parsed);
}
```

## TRATAMENTO DE ERROS E RESILIÊNCIA

### Worker: Retry Strategy
```typescript
const analysisQueue = new Queue('analysis', { connection: redis });

const worker = new Worker('analysis', processAnalysis, {
  connection: redis,
  concurrency: 2,           // processar 2 jobs simultaneamente
  limiter: { max: 5, duration: 60000 },  // max 5 jobs por minuto
});

// Retry: 3 tentativas com backoff exponencial
const job = await analysisQueue.add('analyze', { analysisId, videoUrl }, {
  attempts: 3,
  backoff: { type: 'exponential', delay: 5000 },
  removeOnComplete: 100,    // manter últimos 100 jobs completos
  removeOnFail: 50,
});
```

### Cleanup de Arquivos Temporários
Após cada job (sucesso ou falha), deletar o diretório temporário:
```typescript
finally {
  await fs.rm(tempDir, { recursive: true, force: true });
}
```

### Timeout por Etapa
- Extração (yt-dlp): timeout 30s
- FFmpeg processing: timeout 60s
- Whisper transcription: timeout 30s
- Claude Vision analysis: timeout 45s
- Claude text analysis: timeout 60s
- Total máximo por job: 240s (4 min)

## PONTOS DE ATENÇÃO CRÍTICOS

1. **yt-dlp deve estar instalado globalmente no sistema.** Verificar no boot do worker e falhar fast com mensagem clara se não encontrar.

2. **FFmpeg deve estar instalado globalmente.** Mesmo tratamento.

3. **Limpar SEMPRE os arquivos temporários.** O diretório temp pode encher rápido com vídeos e frames. Usar try/finally no worker.

4. **Cache de análises.** Antes de criar um novo job, verificar se o mesmo videoUrl já foi analisado nas últimas 24h. Se sim, retornar o resultado cacheado imediatamente.

5. **Validar o JSON do Claude.** O LLM pode retornar JSON mal-formado. Implementar: (a) strip de markdown fences, (b) JSON.parse com try/catch, (c) validação Zod, (d) se falhar, retry com prompt mais assertivo.

6. **Controlar custos.** Logar o custo estimado por análise (tokens usados × preço por token). Campo processing_cost_cents no banco.

7. **Rate limiting básico.** No MVP sem auth, implementar rate limit por IP: máximo 5 análises por hora por IP. Usar ioredis com TTL.

8. **CORS.** Configurar @fastify/cors para aceitar requests de localhost:3000 (dev) e do domínio de produção.

9. **Frames para Vision API.** Não enviar todos os frames. Selecionar estrategicamente: frame 1 (gancho), frames nos pontos de corte, frame do meio, último frame. Máximo 10 frames por análise.

10. **Erros amigáveis no frontend.** Nunca mostrar stack traces. Mapear erros do backend para mensagens amigáveis em português.
